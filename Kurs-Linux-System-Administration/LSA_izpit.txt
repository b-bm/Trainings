1.  Поради някаква причина има проблем с конфигурацията на Grub-а, оправете я.
2.  Нямате root парола за този linux, сменете root паролата
3.  Направете файл /fs1 с големина 1GB
4.  Направете файлова система ext3 във файлът /fs1
5.  Монтирайте файловата система от /fs1 в /db
6.  Инициализирайте MySQL база данни в /db, СЛЕД като сте монтирали /fs1 вътре.
7.  Конфигурирайте MySQL сървъра да работи с тази директория(/db)
8.  Сменете shell-а на потребител news на /sbin/nologin
9.  Направете системен потребител wp1
10. Изтриите системен потребител JJtest
11. Променете home folder-а на user mysql на /db
12. Направете база данни wp1
13. Направете mysql потребител wp1 и му дайте пълни права върху базата данни wp1. Паролата му трябва да бъде 'okmijn'.
14. Стартирайте Apache
  Hint: след като го стартирате, може да използвате документацията му: http://localhost/manual/
15. Конфигурирайте Apache така, че да започне да обработва PHP-та.
16. Разархивирайте /root/wordpress.tar.gz в директория /home/wp1/public_html/wordpress
17. Конфигурирайте Apache-а с mod_userdir или VirtualHost така, че да може да отворите /home/wp1/public_html/wordpress през web
18. Конфигурирайте IP 10.8.0.___/24 на eth0, добавете default gw 10.8.0.1 и настройте 8.8.8.8 за DNS
19. Пуснете SSH service-а.
 

След reboot се проверява резултата... ако нещо не е направено да изкара reboot, не се смята за свършено. Точки за смяната на root паролата не се получават.
